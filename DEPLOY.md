# Deployment Guide — Ansar Mahmood

This repo powers **https://ansarmahmood.org** and is served from Hostinger
with **Git auto-deploy** — every push to `main` automatically pulls onto
the server into `public_html/`.

---

## Repo layout

```
Ansar/  (= public_html/ on Hostinger)
│
├── ansar-next/          React SOURCE (Next.js 16, TypeScript, Tailwind v4)
│   ├── src/             components, pages, styles
│   ├── public/          static assets served by React
│   ├── scripts/deploy.sh
│   └── package.json
│
├── _next/               BUILT React bundles  ← generated by `npm run deploy`
├── images/              photos & logos       ← copied from ansar-next/public/images
├── index.html           OLD PHP frontend (will be replaced by React later)
├── pages/               OLD PHP frontend (HTML + PHP includes)
├── assets/              OLD PHP frontend (CSS, JS)
│
├── forms/               PHP form handlers — KEEP
├── admin/               PHP admin panel  — KEEP
├── data/                runtime data (CSV logs, settings.json) — KEEP
├── includes/            mailer.php, cms-data.php — KEEP
├── lib/                 PHPMailer — KEEP
├── exam-*.php           exam portal — KEEP
├── theme.php            theme endpoint — KEEP
└── .htaccess            Apache rewrites — KEEP
```

The React frontend and PHP backend **share the same domain**, so React's
`fetch('/forms/contact-handler.php')` goes straight to the live PHP handler
without any proxy configuration.

## Day-to-day development

```bash
cd ansar-next
npm install              # first time only
npm run dev              # → http://localhost:3000
```

Edit files in `ansar-next/src/`. The dev server hot-reloads.

To test form submissions locally, make sure the old PHP site is also running
on XAMPP — the dev site posts to `http://localhost/Ansar/forms/*.php` when
`hostname === 'localhost'`.

## Deploying to production

### Current (staged) deployment

Because only the homepage exists in React, the deploy script runs in
**Stage 1 mode** — it builds React and copies only the `_next/` bundle
to the repo root. The old PHP `index.html` and `pages/*.html` continue
to serve the rest of the site.

```bash
cd ansar-next
npm run deploy           # builds + copies _next/ to repo root
cd ..
git status               # review what changed
git add .
git commit -m "deploy: rebuild React frontend"
git push                 # Hostinger auto-pulls within ~30s
```

### Future (full switch)

When all 22 pages are built in React:

1. Edit `ansar-next/scripts/deploy.sh`:
   - Comment out the Stage 1 line that copies only `_next/`
   - Uncomment the Stage 2 line that copies `out/.`
   - Uncomment the `rm -f ${REPO_ROOT}/index.html` cleanup line
2. Delete the old PHP frontend files in one commit:
   ```bash
   git rm index.html robots.txt sitemap.xml
   git rm -r pages/ assets/css/ assets/js/layout.js
   # Keep: assets/images/ (React references these paths too)
   ```
3. Run `npm run deploy`, commit, push.

## Emergency rollback

If a deploy breaks the live site:

```bash
git revert HEAD          # creates a new commit undoing the last one
git push                 # Hostinger pulls the revert
```

Or go back to a specific working commit:

```bash
git log --oneline        # find the last good commit hash
git revert <bad-hash>..HEAD
git push
```

## Things NOT tracked in git (see `.gitignore`)

- `ansar-next/node_modules/` — reinstall with `npm install`
- `ansar-next/.next/` — dev server cache
- `ansar-next/out/` — build output (only its contents get committed, at repo root)
- `data/leads.csv`, `data/bookings.csv`, `data/subscribers.csv` — runtime logs
- `data/mail-errors.log` — runtime logs
- `.claude/settings.local.json` — Claude Code local harness state
- `.env*` — any environment secrets

**`data/settings.json` IS tracked** — so admin-panel setting changes need to
be committed and pushed after editing via the admin UI. If you want per-env
settings, uncomment the `data/settings.json` line in `.gitignore`.
